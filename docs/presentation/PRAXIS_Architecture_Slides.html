Memory, compared

Two takes on agent memory

Both answer the same instinct — remember what's worth reusing — but with opposite trust models: Claude lets the model self-curate; PRAXIS inserts a human gate and a retrieval pipeline.

Claude auto-memorymodel-curated · self-trusting PRAXIS memoryhuman-gated · retrieval-ranked
Captured by The model, mid-session — writes a note when it judges something durable and not already obvious in the codebase. Ingestors distill raw artifacts (prose, images, PRs, session narratives) into typed facts.
Selection gate None — the model writes straight to file; selectivity is judgment only. Human gate — every fact lands proposed; a reviewer promotes proposed → active before it's retrievable.
Stored as Flat markdown, one file per topic + a MEMORY.md index. Per-repo, machine-local, single user. Vector-indexed knowledge graph: facts + provenance, scope/category, org/user multi-tenant.
Recalled by MEMORY.md injected at session start (first 200 lines / 25 KB); topic files read on demand by model judgment — no search. Filtered similarity search: embeddings + floor→relative→cap cutoff, trust weighting, recency decay.
Change over time Model edits or overwrites the file. No contradiction detection or state history. Dedup + conflict detection + supersession; bi-temporal validity windows give point-in-time recall.

PRAXIS trades Claude's zero-friction self-curation for an auditable, multi-user knowledge loop — the human gate and ranked retrieval are the price of trust at team scale.
