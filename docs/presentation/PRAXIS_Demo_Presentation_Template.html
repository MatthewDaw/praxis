Pillar 1 · Architecture & Eval

A measured, self-improving loop

Sessions and PRs stream into a DynamoDB log store; a trigger fires LLM distillation. Human-approved knowledge lands in the Knowledge Graph and is read back into future sessions via a get-context tool call.

raw logs → distill → consolidate / dedup → score → human gate → knowledge graph → injected next session → measure ↺

Eval harness — the spine

Fixed tasks on a deliberately quirky repo, run cold vs. knowledge-injected. Metrics: corrections, repeated failures, tokens, time, success rate — so we prove compounding gains, not assert them.

Pillar 3 · ML & Knowledge Pipeline

From raw logs to scored knowledge

  • Learning-moment detection: user corrections, failure→fix transitions, explicit preferences — heuristics, a trained ML classifier, and LLM tagging.
  • LLM distillation turns each into a structured candidate: {when-it-applies, lesson, scope, evidence, form}.
  • Every candidate keeps provenance — a link to the exact log line.

Extractive vs. abstractive

How a lesson is phrased is a first-class experiment — research favors extractive for faithfulness. Distillation is benchmarked, not assumed.

Consolidate · Score · Decay

Deduped, scored, contradiction-checked

  • Cluster & dedup with embeddings + HDBSCAN, so the same lesson isn't learned twice.
  • Confidence score from evidence frequency, recency, and breadth.
  • Contradiction detection surfaces conflicts for the human gate; decay ages out stale lessons.

Into the Knowledge Graph

Scored, deduped, human-approved lessons land in the graph with provenance preserved — ready for just-in-time retrieval in the next session.
