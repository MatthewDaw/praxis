The Problem & The Loop

Agents re-learn the same things

Claude Code's auto-memory is a black box: no quality gate, no dedup, no measurement, per-repo only. Every session's full JSONL transcript — the real record of mistakes, corrections, and fixes — is discarded as exhaust.

PRAXIS makes it a compounding asset.

The knowledge loop

raw logs → extract lessons → consolidate / dedup → confidence score → human gate → knowledge graph → injected next session → measure ↺
Pillar 2 · The Human Gate

Knowledge only ships if a human approves it

  • Promotion lifecycle: proposed → suggested → active → decayed.
  • Each candidate shows its confidence score and links to the exact transcript line (provenance).
  • One click promotes a credible lesson; contradictions are surfaced for human resolution.
  • [[ Speaker note: this is the trust layer — prevents "negative compounding". ]]

Why The Gate Matters

Transparent, measurable knowledge promotion

The central risk is negative compounding — one wrong, over-generalized lesson degrades every future session.

  • Human gate + provenance + thresholds + decay all exist to keep the well clean.
  • Credibility metrics make promotion visible and auditable.
  • [[ Hand-off line to Speaker 3: "…and here's how we prove it works." ]]
Pillar 3 · Architecture & Eval

Built on a measurement spine

  • Sessions + PRs stream into a DynamoDB log store; a trigger fires distillation.
  • Approved knowledge lands in the Knowledge Graph.
  • Future sessions pull context via a get-context tool call.

Eval harness

Fixed tasks on a deliberately quirky repo, run cold vs. knowledge-injected. Metrics: corrections, repeated failures, tokens, time, success rate.

Wrap-up

The agent makes a mistake once — never again

PRAXIS turns discarded logs into a flywheel: the more you use the agent, the better the shared knowledge substrate gets — and we measure it, not assert it.
