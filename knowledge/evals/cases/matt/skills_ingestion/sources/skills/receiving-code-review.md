# receiving-code-review

Evaluate code-review feedback with technical rigor before implementing — avoid
performative agreement and blind implementation.

## When to use
- You receive review comments on your pull request.
- Feedback seems unclear or potentially problematic.
- You need to prioritize multiple review items.
- You want to push back on a suggestion using technical reasoning.

## Workflow
Verify feedback against actual codebase behavior and tests before acting. Ask for
clarification on ambiguous items rather than guessing. Use technical reasoning to
challenge suggestions that would break functionality or violate established
principles. Implement multi-item feedback in priority order (blocking issues
first), testing each fix individually. Acknowledge correct feedback by describing
the specific change made, not with generic gratitude.
