# test-driven-development

Write tests first, watch them fail, then implement the minimal code to pass.

## When to use
- New features, bug fixes, refactoring, behavior changes.

## Workflow
Red-green-refactor. Write a failing test to establish the requirement. Verify the
test fails correctly — this proves it validates the intended behavior ("If you
didn't watch the test fail, you don't know if it tests the right thing"). Write
only the minimal code to pass. Refactor while keeping tests green. Addresses
common workarounds (testing afterward, manual verification, sunk-cost reasoning)
with rebuttals and a verification checklist.
